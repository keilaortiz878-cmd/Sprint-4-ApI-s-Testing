Sprint 4: API Testing with Postman

## 📋 Project Overview
Comprehensive API testing project focusing on Urban Grocers delivery service endpoints. Analyzed requirements for "Working with kits" and "Working with deliveries" features, designed 72 test cases, and executed them using Postman.

## 🎯 Key Features Tested
- **Working with Kits**: Kit creation, modification, and validation
- **Working with Deliveries**: Delivery scheduling, tracking, and management

## 📊 Testing Results Summary
- **Total Test Cases**: 77
- **Passed**: 36
- **Failed**: 41
- **Bug Reports Created**: 41

## 🛠️ Tools & Technologies
- **Postman**: API testing and validation
- **Jira**: Bug tracking and reporting
- **API Documentation**: Endpoint analysis and requirements review

## 📈 Skills Demonstrated
- API requirement analysis and decomposition
- Test case design using equivalence partitioning and boundary value analysis
- Postman collection creation and execution
- Professional bug reporting for API issues
- Test coverage analysis and reporting
